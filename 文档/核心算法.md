在本系统中，**“核心算法”**并非单一的传统函数，而是一套由**密码学原语**、**零知识电路逻辑**和**链上验证逻辑**组成的复合体系。

我们将系统中的算法分为三个层级：

1. **基础密码学算法层**：负责签名验证和哈希计算（确保数据真实）。
2. **零知识证明电路层 (Circom)**：系统的核心大脑，负责在加密状态下执行健康逻辑判断（确保隐私）。
3. **链上验证算法层 (Solidity)**：负责低成本地验证数学证明（确保执行可信）。

以下是详细的算法清单与说明：

------

### **1. 🛡️ 基础密码学算法：EdDSA (Ed25519) 签名验证**

#### ***\*功能\****

用于验证健康数据的**来源真实性**。确保输入到系统中的身高、体重等数据是由可信机构（医院）签发的，且未被用户或黑客篡改。

#### ***\*设计原理\****

- **非对称加密**：机构持有私钥 ( SK*S**K* )，系统公开公钥 ( PK*P**K* )。
- **椭圆曲线**：基于 Curve25519 曲线，相比 ECDSA (secp256k1)，EdDSA 在零知识电路中的约束数量更少（约减少 10 倍），能显著降低生成 Proof 的时间和 Gas 费。
- **确定性签名**：相同的消息和私钥永远生成相同的签名，防止随机数泄露导致的私钥破解。

#### ***\*使用方法\****

1. **签发阶段**：医院计算 Sig=Sign(SKinst,Hash(Data))*S**i**g*=*S**i**g**n*(*S**K**in**s**t*,Hash(*D**a**t**a*)) 。
2. **验证阶段**：在 Circom 电路中，输入 Data,Sig,PK*D**a**t**a*,*S**i**g*,*P**K* ，算法输出 `true` 当且仅当签名合法。

#### ***\*核心代码 (Circom 实现)\****

Circom 标准库 `circomlib` 提供了现成的组件，无需手写底层数学。

```
// 文件名: signature_check.circom
include "circuits/eddsa.circom";
include "circuits/smt_verifier.circom"; // 如果需要默克尔树验证

template EdDSAVerifierComponent(nInputs) {
    // 实例化 EdDSA 验证器
    component eddsa = EdDSAVerifier(nInputs);
    
    // 输入信号定义
    signal input R8[2];      // 签名的一部分
    signal input S[1];       // 签名的另一部分
    signal input A[2];       // 公钥 (Public Key)
    signal input M[nInputs]; // 被签名的消息 (如：身高，体重)

    // 连接信号
    eddsa.R8 <== R8;
    eddsa.S <== S;
    eddsa.A <== A;
    for (var i = 0; i < nInputs; i++) {
        eddsa.M[i] <== M[i];
    }

    // 输出：1 表示验证通过，0 表示失败
    signal output out;
    out <== eddsa.out;
}

// 使用示例：验证包含 2 个数据项的签名
component verifier = EdDSAVerifierComponent(2);
```

### **2. 🧠 核心业务逻辑算法：健康约束电路 (Health Constraint Circuit)**

这是本系统的**灵魂算法**。它将传统的 `if-else` 逻辑转化为**算术电路约束**。

#### ***\*功能\****

在**不泄露具体数值**的前提下，判断用户健康指标是否满足特定规则（如：BMI 正常、血压达标、无特定病史）。

#### ***\*设计原理\****

- **算术化 (Arithmetization)**：将逻辑判断转化为多项式方程。例如， A<B*A*<*B* 转化为存在一个非负整数 k*k* 使得 B−A−1=k*B*−*A*−1=*k* (在有限域内处理需特殊技巧，通常使用范围证明)。
- **范围证明 (Range Proof)**：确保输入数据在合理范围内（如身高不可能为负数或 100 米），防止溢出攻击。
- **组合约束**：将多个条件（AND/OR）串联，只有所有条件满足，电路才能输出 `1`。

#### ***\*使用方法\****

1. 定义私有输入（用户数据）和公共输入（阈值）。
2. 编写 `.circom` 文件描述逻辑。
3. 编译生成 `.wasm` (用于本地计算) 和 `.zkey` (用于生成证明)。

#### ***\*核心代码 (Circom 实现)\****

这是一个完整的 BMI 判断电路示例，包含了签名验证和业务逻辑。

```
// 文件名: health_policy.circom
pragma circom 2.0;
include "circuits/eddsa.circom";
include "circuits/comparators.circom"; // 引入比较器库

template HealthPolicyCircuit() {
    // --- 1. 输入定义 ---
    // 机构签名相关
    signal input R8[2];
    signal input S[1];
    signal input issuerPubKey[2]; 
    
    // 用户健康数据 (私有输入，绝不公开)
    signal input height_cm;   // 身高 (cm)
    signal input weight_kg;   // 体重 (kg) * 100 (为了保持精度，放大100倍)
    signal input timestamp;   // 时间戳

    // 公共输入 (链上可见)
    signal input expected_result; // 期望结果 (1=通过)
    signal input max_bmi_scaled;  // 最大允许 BMI * 100 (例如 2400 代表 24.0)

    // --- 2. 组件实例化 ---
    
    // A. 验证机构签名 (确保数据没被篡改)
    // 消息包括：height, weight, timestamp
    component eddsaVerifier = EdDSAVerifier(3);
    eddsaVerifier.R8 <== R8;
    eddsaVerifier.S <== S;
    eddsaVerifier.A <== issuerPubKey;
    eddsaVerifier.M[0] <== height_cm;
    eddsaVerifier.M[1] <== weight_kg;
    eddsaVerifier.M[2] <== timestamp;

    // B. 范围检查 (确保数据合理：身高 50-250cm, 体重 20-300kg)
    // GreaterEqThan 返回 1 如果 A >= B
    component heightMin = GreaterEqThan(10); 
    heightMin.in[0] <== height_cm;
    heightMin.in[1] <== 50; 

    component heightMax = LessThan(10);
    heightMax.in[0] <== height_cm;
    heightMax.in[1] <== 251;

    // C. 核心业务逻辑：BMI 计算与判断
    // 公式：BMI = weight(kg) / (height(m))^2
    // 电路中使用整数运算：BMI_scaled = (weight * 100) * 10000 / (height^2)
    // 简化逻辑：weight * 10000 <= max_bmi * height * height
    
    signal height_sq;
    signal weight_scaled_limit;
    
    height_sq <== height_cm * height_cm;
    // 注意：Circom 中除法较复杂，通常转换为乘法约束
    // 我们要证明：(weight_kg * 10000) / (height_cm * height_cm) <= max_bmi_scaled
    // 等价于：weight_kg * 10000 <= max_bmi_scaled * height_sq
    
    weight_scaled_limit <== max_bmi_scaled * height_sq;
    
    component bmiCheck = LessEqThan(64); // 64 bit 足够大
    bmiCheck.in[0] <== weight_kg * 10000; 
    bmiCheck.in[1] <== weight_scaled_limit;

    // --- 3. 最终约束输出 ---
    signal output result;
    
    // 逻辑：签名有效 AND 身高合理 AND BMI 达标
    // 在电路中，布尔值相乘即为 AND 操作
    result <== eddsaVerifier.out * heightMin.out * heightMax.out * bmiCheck.out;

    // 强制约束：计算出的 result 必须等于公共输入中的期望值
    result === expected_result;
}

component main = HealthPolicyCircuit();
```

#### ***\*关键算法细节说明\****

- **精度处理**：电路不支持浮点数。我们将体重放大 100 倍，BMI 阈值放大 100 倍，通过乘法代替除法来避免精度丢失和复杂的逆元计算。
- **比较器**：`LessThan` 和 `GreaterEqThan` 是 Circom 库中基于二进制分解（Binary Decomposition）实现的复杂组件，它们将数字拆分为比特位逐位比较。

### **3. ⛓️ 链上验证算法：Groth16 验证器 (Solidity)**

#### ***\*功能\****

在区块链上以极低的 Gas 成本验证用户提交的 Proof 是否合法。

#### ***\*设计原理\****

- 配对检查 (Pairing Check)：利用椭圆曲线配对（Elliptic Curve Pairing, e(P,Q)*e*(*P*,*Q*)）的性质。
  - 验证公式简化版： e(A,B)⋅e(C,D)=e(E,F)*e*(*A*,*B*)⋅*e*(*C*,*D*)=*e*(*E*,*F*) 。
  - 如果等式成立，说明证明者确实知道满足电路约束的私有输入。
- **常数时间复杂度**：无论电路有多复杂（包含多少个逻辑判断），链上验证的计算量几乎是恒定的（约 250k Gas），非常适合大规模应用。

#### ***\*使用方法\****

1. 使用 `snarkjs` 将 `.zkey` 文件导出为 Solidity 合约 (`verifier.sol`)。
2. 部署该合约到区块链。
3. 业务合约调用 `verifyProof(_pA, _pB, _pC, _publicInputs)`。

### **4. 🔄 辅助算法：Poseidon 哈希**

#### ***\*功能\****

在电路内部对数据进行哈希处理（如生成数据指纹、构建默克尔树），替代传统的 SHA-256。

#### ***\*设计原理\****

- **ZK-Friendly**：SHA-256 在电路中需要数万个约束（非常慢），而 Poseidon 基于有限域算术设计，仅需几百个约束。
- **安全性**：基于海绵结构 (Sponge Construction)，抗碰撞性足以满足 ZK 场景。

#### ***\*使用方法\****

用于在电路中压缩长数据，或验证数据是否属于某个默克尔树（白名单验证）。

#### ***\*核心代码 (Circom)\****

```
include "circuits/poseidon.circom";

template HashData() {
    component poseidon = Poseidon(3); // 输入 3 个元素
    signal input in[3];
    signal output out;

    poseidon.inputs[0] <== in[0];
    poseidon.inputs[1] <== in[1];
    poseidon.inputs[2] <== in[2];
    out <== poseidon.out;
}
```

### **💡 总结：算法如何协同工作？**

1. **输入端**：**EdDSA 算法**像门卫一样，先检查“通行证”（机构签名）是否有效。
2. **计算端**：**健康约束电路**像黑盒法官，在看不见具体数值的情况下，利用算术约束计算“是否健康”。
3. **输出端**：**Groth16 算法**将上述所有过程压缩成一张“数学彩票”（Proof）。
4. **验证端**：区块链上的**配对检查算法**只需刮开彩票一角（极低算力），就能确信整个过程没有作弊。

这套算法组合实现了**“数据可用不可见”**与**“来源可信可查”**的完美平衡，是本系统技术壁垒的核心所在。
